# EmergencyRND V2 — Сцена и 3D модели

## Scene Layout

### Building (clinic interior)
- Размер: 16x12 (x: -8..8, z: -12..0)
- Стены толщиной 0.2, высота 3.0
- Вход: южная стена (z=0), проём шириной 2.4 (x: -1.2..1.2)
- Потолок только над зданием (y=3.05)
- Пол: процедурная плитка (Canvas texture)
- Освещение: сетка PointLight + fixture-боксы (флуоресцентный стиль)

### Furniture
- **Ресепшн-стойка** — по центру (z=-9.0), составная: столешница + передняя панель (z=-8.5) + задняя панель (z=-9.5) + боковины (x: -1.5, 1.5)
- **Касса** — справа от ресепшна (x=3.5, z=-9.5): деревянный стол + терминал оплаты банковской картой (корпус + экран + клавиатура)
- **3 стула (начальные)** — вдоль восточной стены (x=6.5, z: -2, -3.2, -4.4), лицом к центру, rotY = -π/2. Управляются системой `Game.Furniture`
- **2 медицинские кровати (начальные)** — у западной стены (x=-5.5, z: -7, -9), каркас + матрас + подушка + спинка, rotY = 0. Управляются системой `Game.Furniture`
- **Стиральная машина** — в северо-восточном углу (x=5.5, z=-10.5): белый корпус (0.9×1.0×0.7) + дверца-барабан + индикатор состояния + прогресс-бар. Управляется системой `Game.WashingMachine`
- **2 стеллажа** — у задней стены, рядом друг с другом (x=-5.5 и x=-4.2, z=-11.5), 3 полки каждый, 9 слотов на стеллаж
- Все объекты добавлены в `collidables[]`
- Кровати и стулья — **динамические**: можно покупать в магазине, перемещать клавишей E, ставить в любое место внутри здания
- При создании через `world.js` меши регистрируются в `Game.Furniture` через `registerExisting()`
- Каждый предмет мебели имеет `{ group, collisionBox, slot: {pos, occupied}, isIndoors, isDirty, dirtyOverlay }`
- Кровати имеют свойство `isDirty` — грязное бельё после пациента (визуальный overlay на матрасе)

## Collision System
- Raycasting по 2 осям (X, Z) независимо — скольжение вдоль стен
- Рейкаст игрока идёт от `y=0.5` (центр тела), а не от камеры (y=1.6) — попадает во все collision-боксы мебели
- `collisionDistance = 0.4`
- Массив `collidables[]` содержит: стены, полы (indoor/outdoor/grass), потолок, мебель, невидимые collision-боксы, дверные стойки, стеллажи, панель инструментов, стол кассы
- Деревья имеют collision-цилиндры
- Пациенты **не** добавлены в `collidables[]` (игрок проходит сквозь них)
- Препараты используют `collidables[]` для физики (столкновения при падении/бросках по осям X, Y, Z)

## 3D Models

### Patient 3D Model
- Иерархическая модель с суставами для анимации:
```
root Group (patient.mesh) — позиция/поворот для движения
  poseContainer (Group) — rotation.x для позы лёжа, position.y для высоты кровати
    bodyContainer (Group) — position.y для позы сидя
      Body: BoxGeometry(0.4, 0.7, 0.25), случайный цвет из BODY_COLORS, y=0.85
      Head: SphereGeometry(0.15), skin color (#f0c8a0), y=1.35
      Left Arm:
        shoulderPivot (Group, x=-0.28, y=1.15) — вращение плеча
          upperArm: BoxGeometry(0.09, 0.24, 0.09), skin color, y=-0.12
          elbowPivot (Group, y=-0.24) — вращение локтя
            forearm: BoxGeometry(0.08, 0.22, 0.08), skin color, y=-0.11
      Right Arm: (зеркально, x=+0.28)
    leftLegPivot (Group, x=-0.1, y=0.5) — вращение бедра
      leftLeg: BoxGeometry(0.14, 0.5, 0.18), тёмные (#334455), y=-0.25
    rightLegPivot (Group, x=+0.1, y=0.5)
      rightLeg: BoxGeometry(0.14, 0.5, 0.18), y=-0.25
```
- `userData.bodyParts = [body, head, leftUpperArm, leftForearm, rightUpperArm, rightForearm, leftLeg, rightLeg]` — для подсветки и raycasting
- `userData` хранит ссылки: `poseContainer`, `bodyContainer`, `bodyMesh`, `headMesh`, `leftArm` (`{shoulderPivot, elbowPivot, upperArm, forearm}`), `rightArm`, `leftArmPivot`, `rightArmPivot`, `leftLegPivot`, `rightLegPivot`

### Bed Indicator (Sprite)
- При переходе пациента в `atBed` создаётся `THREE.Sprite` с Canvas-текстурой над головой
- Canvas 64x64: тёмный круг с цветной обводкой + иконка конкретного предмета (`drawIndicatorIcon()`)
  - При `needsDiagnosis`: иконка нужного инструмента (фонендоскоп/молоток/риноскоп)
  - После диагностики: иконка нужного препарата (блистер/флакон/коробка с крестом)
- SpriteMaterial с `transparent: true`, `depthTest: false`
- Scale: базовый `(0.4, 0.4, 1)`, пульсирует ±0.06 (sin-анимация), позиция y=2.0 (стоя) / y=1.5 (лёжа)
- Хранится в `patient.indicator`, удаляется при лечении
- При `revealDiagnosis()` индикатор пересоздаётся (инструмент → препарат)

### Consumable 3D Models (препараты)
```js
CONSUMABLE_TYPES = {
  strepsils:     { name: 'Стрепсилс',        color: 0xcc3333, size: {x:0.15, y:0.08, z:0.10} },
  painkiller:    { name: 'Обезболивающее',    color: 0x3366cc, size: {x:0.18, y:0.06, z:0.12} },
  antihistamine: { name: 'Антигистаминное',   color: 0x33aa55, size: {x:0.14, y:0.07, z:0.10} },
  linen_clean:   { name: 'Постельное бельё',  color: 0xeeeeff, size: {x:0.20, y:0.10, z:0.15} },
  linen_dirty:   { name: 'Грязное бельё',     color: 0x887766, size: {x:0.20, y:0.10, z:0.15} }
}
```
Каждая модель — `THREE.Group` с несколькими примитивами:
- **Стрепсилс** — блистер: BoxGeometry основа + 6 SphereGeometry полусфер (2×3) + фольга снизу
- **Обезболивающее** — флакон: CylinderGeometry тело + белая крышка + белая этикетка-полоса
- **Антигистаминное** — коробочка: BoxGeometry + белый крестик "+" на передней грани
- **Постельное бельё (чистое)** — сложенная простыня: BoxGeometry(0.20, 0.08, 0.15), светло-голубой + линия сгиба + акцентная полоса
- **Грязное бельё** — сложенная простыня: BoxGeometry(0.20, 0.08, 0.15), коричневатый + пятна загрязнений
- `group.userData.consumableType = type`

### Dirty Bed Overlay
- При грязном белье (`isDirty = true`) на матрас кровати добавляется overlay
- BoxGeometry(1.9, 0.02, 0.8), полупрозрачный коричнево-жёлтый (0xaa8844, opacity 0.5)
- Позиция y=0.63 (чуть выше матраса), дочерний элемент группы кровати
- Создаётся/удаляется через `setDirty(item, true/false)`

### Washing Machine 3D Model
- Позиция: (5.5, 0, -10.5), северо-восточный угол здания
- **Корпус**: BoxGeometry(0.9, 1.0, 0.7), белый/серый (0xdddddd), y=0.5
- **Верхняя панель**: BoxGeometry(0.9, 0.06, 0.7), чуть темнее (0xcccccc), y=1.03
- **Дверца-барабан**: CylinderGeometry(r=0.22, h=0.02), тёмный (0x333344), z=0.36, y=0.45. Вращается при стирке
- **Обод двери**: TorusGeometry(R=0.23, r=0.02), серый, z=0.36, y=0.45
- **Панель управления**: BoxGeometry(0.6, 0.12, 0.02), серый, z=0.36, y=0.85
- **Индикатор состояния**: SphereGeometry(r=0.03), зелёный emissive (свободна), жёлтый (стирка), оранжевый (загружено)
- **Прогресс-бар**: два меша (фон + заполнение) над дверцей, видимы только при стирке
- Collision box (1.0, 1.1, 0.8) в `collidables[]`
- Табличка "СТИРКА" на задней стене (5.5, 2.5, -11.78)

### Box 3D Model (коробка препаратов)
- BoxGeometry(0.8, 0.6, 0.6), цвет коробки = цвет препарата (roughness 0.6)
- Тёмная окантовка сверху (цвет × 0.7)
- Canvas-текстура 512×512 на передней И задней гранях: белая этикетка, цветной мед. крест, название (52px bold), "×10" (72px bold), "препараты"/"расходники" (32px)
- Надписи: `PlaneGeometry(0.68, 0.51)`, offset +0.005 от грани, `polygonOffset: true`, `userData.isLabel = true` (пропускаются при highlight)
- `group.userData.isBox = true`, `group.userData.boxType = type`

### Cashier Desk 3D Model

### Shelf 3D Model
- `THREE.Group`: задняя панель (1.2x1.5x0.05) + 2 боковины (0.04x1.5x0.4) + 3 полки на y=0.3, 0.7, 1.1
- Материал: дерево (0x8B6F47, roughness 0.6)
- Collision box (1.3x1.5x0.5) в `collidables[]`
- 2 стеллажа рядом: x=-5.5 и x=-4.2, z=-11.5
- **Инструменты не принимаются** — при попытке размещения показывается уведомление "Инструменты вешайте на панель"

### Shelf Slots
```js
{ mesh, highlightParts[], slots: [{ pos: Vector3, item: null, itemMesh: null }] }
```
- 9 слотов на стеллаж (3 на полку × 3 полки)
- На каждой полке 3 позиции: left (x=-0.35), center (x=0), right (x=0.35) в локальных координатах
- `pos` — мировые координаты на поверхности полки

### Tool Panel 3D Model
Настенная панель для инструментов, справа от стеллажей.
- `THREE.Group`: деревянная доска `BoxGeometry(1.0, 1.2, 0.06)` (0x6B5B4F) + рамка из 4 планок (0x4A3B30)
- 3 металлических крюка (CylinderGeometry r=0.02, h=0.12 + SphereGeometry r=0.03 на конце) — цвет 0x888899, metalness 0.6
- Подписи-таблички под каждым крюком (PlaneGeometry 0.4×0.1, Canvas texture с названием инструмента)
- Collision box (1.1×1.3×0.3) в `collidables[]`
- Позиция: x=-3.0, y_bottom=0.5, z=-11.85 (вплотную к северной стене)
- Табличка "ИНСТРУМЕНТЫ" на стене (x=-3.0, y=1.95, z=-11.78)

### Tool Panel Slots
```js
{ type: 'instrument_stethoscope', pos: Vector3, item: null, itemMesh: null, hookMesh: ref }
```
- 3 типизированных слота (каждый под конкретный инструмент):
  - Фонендоскоп: y_offset=0.25 (мир: y=0.75)
  - Рефлекс-молоток: y_offset=0.60 (мир: y=1.10)
  - Риноскоп: y_offset=0.95 (мир: y=1.45)
- Инструменты масштабируются 2.5x и висят с наклоном (rotation.z=0.15)

### Delivery Zone
- Плоская зона на земле перед входом (PlaneGeometry 3x2, y=0.01)
- Жёлтая пунктирная граница, текст "ДОСТАВКА" (Canvas texture)
- Препараты/инструменты спавнятся в случайной точке внутри зоны на высоте y=3 и падают
- Мебель (кровати, стулья) спавнится в зоне доставки на y=0 (без физики падения)
