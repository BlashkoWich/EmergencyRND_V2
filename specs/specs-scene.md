# EmergencyRND V2 — Сцена и 3D модели

## Scene Layout

### Building (clinic interior)
- Размер: 16x12 (x: -8..8, z: -12..0)
- Стены толщиной 0.2, высота 3.0
- Главный вход: южная стена (z=0), проём шириной 2.4 (x: -1.2..1.2)
- Боковая дверь: западная стена (x=-8), проём шириной 2.4 в северном углу (z: -12..-9.6), ведёт к зоне доставки
- Потолок только над зданием (y=3.05)
- Пол: процедурная плитка (Canvas texture)
- Освещение: сетка PointLight + fixture-боксы (флуоресцентный стиль)

### Furniture
- **Ресепшн-стойка** — по центру (z=-9.0), составная: столешница + передняя панель (z=-8.5) + задняя панель (z=-9.5) + боковины (x: -1.5, 1.5)
- **Касса самообслуживания** — слева от входа (x=-3.5, z=-1.5), повёрнута на 180° (rotation.y=Math.PI), экран смотрит вглубь здания. Металлический корпус + наклонная панель с LCD-экраном + card reader + receipt slot. Над кассой — 3D sprite-индикатор (стопка монет / мешочек / большой мешок) отображающий накопленную сумму. Вся касса — единая зона взаимодействия: удержание E снимает деньги (когда они есть) или переносит кассу (когда она пуста)
- **3 стула (начальные)** — вдоль восточной стены (x=6.5, z: -2, -3.2, -4.4), лицом к центру, rotY = -π/2. Управляются системой `Game.Furniture`
- **4 медицинские кровати (начальные)** — у западной стены (x=-5.5, z: -9, -7, -5, -3), каркас + матрас + подушка + спинка, rotY = 0. Управляются системой `Game.Furniture`
- **2 стеллажа** — у задней стены, рядом друг с другом (x=-5.5 и x=-4.2, z=-11.5), 3 полки каждый, 9 слотов на стеллаж. Стартуют пустыми — игрок покупает расходники через магазин
- Все объекты добавлены в `collidables[]`
- Кровати, стулья, стеллажи, касса — **перемещаемые**: можно перемещать зажатием клавиши E (hold 600мс). Кровати и стулья также покупаются в магазине
- При создании через `world.js` меши регистрируются в `Game.Furniture` через `registerExisting()`
- Каждый предмет мебели имеет `{ group, collisionBox, slot: {pos, occupied}, isIndoors }`. Для кроватей дополнительно: `{ hp, frame, rail, frameMat, cracks, brokenDebris }`
- Кровати имеют HP 0..10 (`BED_MAX_HP = 10`) — счётчик износа, уменьшается на 1 после каждого пациента, восстанавливается ремонтным ключом

- **Indoor**: AmbientLight(0.3) + сетка из 9 PointLight (x: -6..6 шаг 4, z: -10..-2 шаг 4). **Тени отключены** на всех PointLight (только освещение)

- Fixture-меши (светильники) — эмиссивные (`MeshLambertMaterial` с `emissive`)

## Collision System
- Raycasting по 2 осям (X, Z) независимо — скольжение вдоль стен
- Рейкаст игрока идёт от `y=0.5` (центр тела), а не от камеры (y=1.6) — попадает во все collision-боксы мебели
- `collisionDistance = 0.4`
- Массив `collidables[]` содержит: стены, полы (indoor/outdoor/grass), потолок, мебель, невидимые collision-боксы, дверные стойки, стеллажи, стол кассы
- Деревья имеют collision-цилиндры
- Пациенты **не** добавлены в `collidables[]` (игрок проходит сквозь них)
- Препараты используют `collidables[]` для физики (столкновения при падении/бросках по осям X, Y, Z)

### Wall Clamping (перемещение мебели)
- Мебель **нельзя перетаскивать через стены** — позиция клампится перед стеной
- Реализовано через 2D line-segment intersection в `wallClampT()` (`furniture.js`)
- Стены определены как осевые сегменты `WALL_SEGMENTS[]`: 5 сегментов (north, west south-of-door, east, south-left, south-right)
- Дверные проёмы (главный вход x: -1.2..1.2 и боковая дверь z: -12..-9.6) не имеют сегментов — мебель проходит свободно
- Отступ от стены: `max(boxSize.x, boxSize.z) / 2 + WALL_HALF_THICKNESS + 0.05`

### Wall-Mount Placement (настенные предметы)
- Предметы с флагом `wallMount: true` при переносе привязываются к ближайшей стене (`getWallPlacementPosition()`)
- Находится ближайший `WALL_SEGMENTS` к позиции курсора на полу (только если предмет помещается в пределах стены по ширине)
- Предмет сдвигается вплотную к стене: offset = `boxSize.z / 2 + WALL_HALF_THICKNESS + 0.01`
- Поворот автоматический (лицом в комнату): north → rotY=0, south → rotY=π, west → rotY=π/2, east → rotY=-π/2
- Вращение колёсиком мыши отключено для wallMount предметов
- Если стена не найдена — красный outline, размещение запрещено

### Bed Indicator (Sprite)
- При переходе пациента в `atBed` создаётся `THREE.Sprite` с Canvas-текстурой над головой
- Canvas 64x64: тёмный круг с цветной обводкой + иконка конкретного предмета (`drawIndicatorIcon()`)
  - При `needsDiagnosis`: иконка нужного инструмента (фонендоскоп/молоток/риноскоп)
  - После диагностики: иконка нужного препарата (блистер/флакон/коробка с крестом)
- SpriteMaterial с `transparent: true`, `depthTest: false`
- Scale: базовый `(0.4, 0.4, 1)`, пульсирует ±0.06 (sin-анимация), позиция y=2.0 (стоя) / y=1.5 (лёжа)
- Хранится в `patient.indicator`, удаляется при лечении
- При `revealDiagnosis()` индикатор пересоздаётся (инструмент → препарат)

### Consumable 3D Models (препараты)
```js
CONSUMABLE_TYPES = {

  painkiller:    { name: 'Обезболивающее',    color: 0x3366cc, size: {x:0.18, y:0.06, z:0.12} },
```

Каждая модель — `THREE.Group` с несколькими примитивами:
- **Стрепсилс** — блистер: BoxGeometry основа + 6 SphereGeometry полусфер (2×3) + фольга снизу
- **Обезболивающее** — флакон: CylinderGeometry тело + белая крышка + белая этикетка-полоса
- **Антигистаминное** — коробочка: BoxGeometry + белый крестик "+" на передней грани
- `group.userData.consumableType = type`

### Bed Wear Visuals
Меняется только каркас кровати (frame + rail). Матрас и подушка остаются нетронутыми. Трещины и пятна размещаются на **вертикальных гранях рамы** (front/back/left/right), чтобы быть видимыми из любой точки — матрас перекрывает верхнюю грань.

- **HP 6-10** (норма): без изменений
- **HP 1-5** (изношенная): выраженные повреждения на каркасе (все меши из материала `0x101010`, пятна — `0x3a2a22`):
  - Передняя грань (z≈0.455): 3 наклонённых трещины (0.9×0.04, 0.7×0.035, 0.5×0.03) с поворотами ±0.28..0.6 и 2 короткие ответвления
  - Задняя грань (z≈-0.455): 2 трещины (0.8×0.035, 0.5×0.03)
  - Левый/правый торцы (x=±1.005): наклонённые трещины с rotation.x ±0.3
  - 2 крупных тёмно-коричневых пятна (BoxGeometry 0.25×0.12, 0.18×0.09) на передней грани
  - 2 отколотых куска (BoxGeometry 0.12×0.08×0.06, 0.1×0.06×0.05) на верхних передних углах
  - Каркас темнее через `frameMat.color.multiplyScalar(0.78)` относительно базового цвета
- **HP 0** (сломана): всё из HP 1-5 плюс:
  - Каркас ещё темнее (`multiplyScalar(0.55)`)
  - `rail` скрыт (`visible=false`)
  - 2 тёмных обломка (`0x555544`) на полу рядом (y≈0.03..0.05)
  - Группа наклонена: `rotation.z = baseZ + 0.06` (~3.4°)
- Визуал пересоздаётся через `applyBedVisual(item)` при каждом изменении `item.hp`. Базовый цвет рамы кешируется в `item._baseFrameColor`, исходный rotation.x/z — в `item._baseRotX/Z`, трещины — в `item.cracks[]`, обломки — в `item.brokenDebris[]`
