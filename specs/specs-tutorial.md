# EmergencyRND V2 — Туториал

## Overview
Пошаговый интерактивный туториал для новых игроков. Запускается **только при выборе уровня 1**. Проводит игрока через все базовые механики: открытие смены, работа с пациентами, магазин, лечение, касса. Финальный шаг рассказывает про износ кроватей и ремонтный ключ (R). Все действия, не относящиеся к текущему шагу, **заблокированы** визуально и технически. Коробки с расходниками расходуются нормально (remaining уменьшается при каждом взятии). Начисление XP во время туториала **отложено** — XP накапливается, но уровень не повышается до завершения туториала (предотвращает софтлок с попапом уровня 2).

## Файл: `js/tutorial.js`

### Архитектура
IIFE-модуль `Game.Tutorial` — конечный автомат (state machine) с массивом шагов `STEPS[]`. Каждый шаг определяет:
- `text` — текст инструкции (через `Game.Lang.t('tutorial.stepN')`)
- `trigger` — событие для перехода к следующему шагу (`'next_button'` / `'shift_opened'` / `'popup_opened'` и т.д.)
- `allowed[]` — список разрешённых действий (всё остальное заблокировано)
- `pauseTime` — останавливать ли игровое время
- `position` — позиция панели (`'center'`, `'left'`, `'top-right'`)
- `spotlight` — CSS-селектор элемента для подсветки (spotlight)
- `needsPointerLock` — нужен ли захват мыши для 3D-взаимодействия
- `arrow3D` — координаты для 3D-стрелки `{x, y, z}`
- `arrowTrackPatient` — стрелка следит за пациентом
- `arrowOffsetY` — высота стрелки над целью
- `onEnter` / `onExit` — коллбэки при входе/выходе из шага
- `isFinal` — последний шаг (кнопка "Начать!")

Все тексты шагов локализованы через `Game.Lang.t('tutorial.stepN')`. Шаги с динамическим HTML используют `Game.Lang.t('tutorial.stepNhtml', [params])`.

### Публичный API
```js
Game.Tutorial.setup(controls, THREE, camera, scene)  // инициализация
Game.Tutorial.checkStart()                            // проверка и запуск (вызывается из levels.js)
Game.Tutorial.isActive()                              // активен ли туториал
Game.Tutorial.isPaused()                              // нужно ли паузить игровое время
Game.Tutorial.isAllowed(actionId)                     // разрешено ли действие на текущем шаге
Game.Tutorial.onEvent(eventName, data)                // модули вызывают при событиях
Game.Tutorial.setTutorialPatient(patient)              // сохраняет ссылку на первого пациента
Game.Tutorial.update(delta)                           // обновление 3D-стрелки (вызывается в animation loop)
```

### Активация
- Вызывается из `levels.js` при выборе уровня 1: `Game.Tutorial.checkStart()`
- Запускается после первого `controls.lock` (когда игрок выбрал уровень 1 на экране выбора уровня)
- **Не сохраняется** в localStorage — запускается при каждом выборе уровня 1

### Блокировка действий
Каждый модуль проверяет `Game.Tutorial.isAllowed(actionId)` перед выполнением действия. Если туториал не активен, `isAllowed()` всегда возвращает `true`.

**Action IDs:**
- `movement`, `camera` — перемещение и камера
- `sign_click` — клик по табличке смены
- `patient_click` — клик по пациенту
- `treat_patient` — применение препарата на пациента
- `btn_bed`, `btn_wait`, `btn_dismiss` — кнопки попапа пациента
- `shop_open`, `shop_close`, `shop_buy` — магазин
- `shelf_take`, `shelf_place` — взятие/размещение на стеллаже
- `pickup_item`, `pickup_box`, `drop_item` — подбор/бросок предметов
- `cashier_click` — клик по терминалу кассы
- `furniture_interact` — перемещение мебели
- `trash_interact` — уборка мусора

## UI элементы

### HTML (`index.html`)
```html
<div id="tutorial-overlay">
  <div id="tutorial-spotlight"></div>
  <div id="tutorial-panel">
    <div id="tutorial-step-num"></div>
    <div id="tutorial-text"></div>
    <button id="tutorial-next" data-lang-key="tutorial.next">Далее</button>
  </div>
</div>
```

### CSS (z-index)
- `#tutorial-spotlight` — z-index: 16 (ниже попапов z:20, выше HUD z:5)
- `#tutorial-panel` — z-index: 30 (выше всех попапов)
- `#tutorial-overlay` — z-index: 28

### Spotlight
Реализован через `box-shadow: 0 0 0 9999px rgba(0,0,0,0.6)` — создаёт тёмный оверлей с прозрачной "дыркой" вокруг целевого элемента.

### Заблокированные кнопки
CSS-класс `.tutorial-locked`:
```css
.tutorial-locked {
  opacity: 0.35 !important;
  pointer-events: none !important;
  filter: grayscale(0.8) !important;
}
```

## 3D-стрелка
Голубая 3D-стрелка (цилиндр + конус + кольцо) из `THREE.MeshBasicMaterial({ color: 0x44bbff })`:
- Парит над целевой точкой с покачиванием (sin-анимация)
- Медленно вращается вокруг оси Y
- Поддерживает статические координаты (`arrow3D`) и динамическое слежение (`arrowTrackPatient`, `arrowTrackBed`)

## Первый бесплатный заказ
В `shop.js` реализована механика: первый заказ каждого типа расходников — бесплатный.
- Словарь `firstOrderUsed{}` — отслеживает использованные бесплатные заказы
- Кнопки показывают "Заказать — Бесплатно!" зелёным цветом
- После использования: обычная цена "Купить — $80/$100"

## Стеллажи
Стеллажи стартуют **пустыми** — игрок покупает расходники через магазин. Предзаполнение удалено.

### Блокировки в модулях:
| Модуль | Что блокируется |
|--------|----------------|
| `shift.js` | Клик по табличке, пауза времени, блокировка спавна |
| `patients.js` | Клик по пациенту, кнопки попапа, HP-decay, спавн по таймеру и после оплаты |
| `shop.js` | Открытие/закрытие, покупки |
| `cashier.js` | Клик по терминалу, кнопка закрытия скрыта во время туториала |
| `shelves.js` | Взятие и размещение предметов |
| `consumables.js` | Подбор (кроме коробок — из коробок можно брать всегда), бросок, E-подъём коробок. Коробки расходуются нормально (remaining--) |

| `trash.js` | Уборка мусора |
| `controls.js` | Overlay не показывается при активном туториале; re-lock при ESC |

### Порядок в animation loop:
```
Game.Interaction.update()    // централизованный рейкаст — ближайший объект побеждает
Game.Cashier.update(delta)
Game.Shelves.update(delta)
Game.Tutorial.update(delta)  // обновление 3D-стрелки
```

### Завершение туториала:
При вызове `finish()`:
1. `active = false` — все блокировки снимаются
2. Убираются UI (overlay, spotlight, стрелка)
3. Разблокируются все кнопки и табы
4. `controls.lock()` — возврат к управлению
5. `Game.Levels.checkDeferredLevelUp()` — проверка отложенного повышения уровня (если XP набрано достаточно, показывается попап уровня 2)
6. `Game.Patients.spawnFirstPatient()` — спавн первого пациента после туториала
