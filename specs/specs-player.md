# EmergencyRND V2 — Игрок, управление, взаимодействие

## Player
- Стартовая позиция: `(0, 1.6, -10)` (за ресепшн-стойкой), лицом к улице (+Z)
- Камера: PerspectiveCamera, FOV 70, near 0.1, far 100
- `rotation.order = 'YXZ'` — совпадает с внутренним euler order PointerLockControls, предотвращает quaternion↔euler drift
- Начальный поворот: `camera.quaternion.setFromEuler(new Euler(0, PI, 0, 'YXZ'))` — лицом к входу/улице

## Controls
- **WASD** — движение (через `e.code`, работает на любой раскладке)
- **Мышь** — вращение камеры (Pointer Lock API, `pointerSpeed = 0.5`)
- **ЛКМ** — взаимодействие (пациент / лечение / подбор препарата / взять из коробки / взять со стеллажа / оплата на кассе / убрать мусор)
- **E (KeyE)** — подобрать коробку в руки (не в инвентарь); **зажатие** E (hold 600мс) — поднять мебель (кровать/стул), **нажатие** E — поставить мебель; положить на стеллаж. Если в руке ремонтный ключ и прицел на кровати — **зажатие** E (hold 3000мс) чинит кровать до 10 HP
- **R (KeyR)** — взять/убрать ремонтный ключ (гаечный ключ в first-person в правом-нижнем углу экрана)
- **Q (KeyQ)** — открыть/закрыть магазин препаратов
- **G (KeyG)** — бросить коробку (если держит) или предмет из активного слота инвентаря (вперёд по камере с физикой)
- **1-6 (Digit1-Digit6)** — выбор слота инвентаря
- **Shift (ShiftLeft)** — ускорение (спринт): скорость `7.0` ед/сек вместо `4.0`
- **Space** — прыжок (начальная скорость `5.0`, гравитация `-12.0`); горизонтальное движение сохраняется в воздухе
- **ESC** — пауза (разблокировка курсора, показ `#pause-screen` если игрок уже вошёл в игру, иначе `#overlay`)
- Скорость движения: `4.0` ед/сек (обычная), `7.0` ед/сек (спринт)
- Высота камеры: `1.6` (базовая, меняется при прыжке)

## Mouse Input
- PointerLockControls используется только для lock/unlock (внутренний mousemove отключён через `disconnect()`)
- **Накопление дельт**: `mousemove` listener копит `_mouseDX`/`_mouseDY` с clamp ±100 за событие
- **Применение 1 раз за кадр**: в `update()` накопленные дельты конвертируются в euler-вращение камеры, затем сбрасываются в 0
- Математика идентична PointerLockControls r164: `euler.y -= dx * 0.002 * pointerSpeed`, pitch clamp ±π/2
- `pointerSpeed = 0.5`
- Синхронизация инпута с кадром обеспечивает плавность на слабых ПК (нет дёрганий между кадрами)

## Controls Internal State
```js
_keys { forward, backward, left, right, sprint, jump }  // состояние клавиш
_raycaster                // для коллизий
_collisionOrigin          // Vector3, точка начала рейкаста (y=0.5, центр тела)
_savedQuat                // Quaternion|null, сохраняется при lock, восстанавливается в первом update
_forward, _right, _moveDir, _moveX, _moveZ  // pre-allocated Vector3, переиспользуются каждый кадр (без GC-давления)
_mouseDX = 0              // накопленная дельта мыши X (сбрасывается каждый кадр в update)
_mouseDY = 0              // накопленная дельта мыши Y (сбрасывается каждый кадр в update)
_mouseEuler               // Euler('YXZ'), переиспользуется для вращения камеры
_moveSpeed = 4.0
_sprintSpeed = 7.0
_collisionDistance = 0.4
_velocityY = 0            // вертикальная скорость (прыжок/падение)
_jumpSpeed = 5.0           // начальная скорость прыжка
_gravity = -12.0           // гравитация
_groundY = 1.6             // базовая высота камеры
_isGrounded = true         // на земле ли игрок
```

## Inventory System

### Data Model
```js
slots = [null, null, null, null, null, null]  // 6 слотов, значение = {type, count} или null
activeSlot = 0  // индекс 0-5
maxStack = 5    // макс. кол-во препаратов в одном слоте (базовое значение)
```

### Стакинг
- Препараты одного типа складываются в один слот (до `maxStack` штук)
- Инструменты НЕ стакаются — всегда занимают отдельный слот (count=1)
- `addItem(type)` для препаратов: сначала ищет существующий стак того же типа с `count < maxStack`, потом пустой слот
- `removeActive()` уменьшает count на 1; если count=0 — слот обнуляется
- На занятых слотах отображается бейдж `.inv-count` с количеством (если count > 1, только для препаратов)
- Базовый maxStack = 5, расширяется через апгрейд "Прокачка героя" в магазине

### UI (`#inventory-container` > `#inventory-active-name` + `#inventory-bar` + `#inventory-hints`)
- Контейнер `#inventory-container`: `fixed bottom: 16px`, по центру, z-index: 5, flex column
- `#inventory-active-name`: название активного предмета над баром (скрыто если слот пуст)
- `#inventory-bar`: 6 слотов (76x76px), rgba(0,0,0,0.5) фон, border
- Активный слот: золотая рамка (#ffcc00) + box-shadow
- Занятые слоты: canvas-иконки 128x128 (уникальные для каждого типа: блистер, флакон, коробка с крестом, фонендоскоп, молоток, риноскоп) — функция `generateIcon(type)` с кешированием
- Бейдж количества `.inv-count`: абсолютно позиционирован в правом верхнем углу слота, показывает count (только для препаратов с count > 1)
- Подпись 1-6 под каждым слотом
- Клавиши Digit1-Digit6 переключают activeSlot
- Поддерживает как препараты (`Game.Consumables.TYPES`), так и инструменты (`Game.Consumables.INSTRUMENT_TYPES`)
- `#inventory-hints`: строка подсказок под слотами
  - "Q — Магазин" — всегда видна
  - "R — Ремонтный ключ" — всегда видна (ключ локализации `inv.wrench`)
  - "G — Бросить" — видна только когда в активном слоте есть предмет
  - Стиль подсказок: крупный шрифт (0.9rem), яркий текст (85% opacity), жирный, тёмный фон

### Notification (`#notification`)
- z-index: 15, по центру
- Поддерживает цвет фона через параметр `color`
- Зелёный (`rgba(34, 139, 34, 0.85)`) для позитивных событий
- Красный (`rgba(200, 50, 50, 0.85)`) по умолчанию для негативных/нейтральных
- Появляется на 2 секунды, затем затухает (opacity transition)

### Inventory API
- `addItem(type)` → boolean (true если добавлено). Для препаратов: сначала стакает в существующий слот, потом в пустой
- `addItemBulk(type, maxCount)` → number (сколько реально добавлено, 0 если не удалось). Добавляет до `maxCount` предметов за раз в один слот (не больше `maxStack`). Для инструментов — максимум 1. Сначала ищет существующий стак, потом пустой слот
- `removeActive()` → type или null. Уменьшает count на 1, обнуляет слот при count=0
- `removeActiveN(n)` → type или null. Уменьшает count на `n`, обнуляет слот при count≤0
- `getActive()` → type активного слота или null
- `getActiveCount()` → number — количество предметов в активном слоте (0 если пуст)
- `findAndActivate(type)` → boolean. Ищет предмет данного типа в инвентаре, переключает activeSlot на найденный. Возвращает true если найден
- `findAndActivateOneOf(types)` → type или null. Ищет первый подходящий из массива типов, переключает activeSlot. Возвращает найденный тип или null
- `isFull()` → boolean
- `countType(type)` → number — суммарное количество предметов данного типа (сумма count по всем слотам)
- `setMaxStack(n)` → устанавливает максимальный размер стака
- `getMaxStack()` → текущий максимальный размер стака
- `showNotification(text, color)` → показать временное уведомление (color опционален, по умолчанию красный)

## Shelf Interaction
- Два режима: **размещение** (place) и **взятие** (take)
- Блокируется если игрок держит коробку (`isHoldingBox()`) или наведён на коробку (`hasBoxInteraction()`)
- **Инструменты нельзя класть на стеллаж** — при попытке показывается уведомление "Инструменты вешайте на панель"

### Размещение (place)
- Raycaster пересекает все меши стеллажей (`allShelfParts[]`)
- Активируется если в активном слоте инвентаря есть **расходник (не инструмент)** И на стеллаже есть доступный слот
- Стекирование: до 10 предметов одного типа на один слот (`MAX_SHELF_STACK = 10`)
- При размещении: сначала ищется слот с тем же типом (для стека), затем пустой слот
- Зелёная обводка (Game.Outline) на всём стеллаже, подсказка "Положить на E"
- **E (KeyE)** → выкладывает весь активный слот bulk-ом: цикл по `getActiveCount()`, каждый предмет размещается через `findSlotForType` + `placeItemOnShelf`, затем `removeActiveN(placed)`. Если стеллаж заполнится раньше — выложит сколько влезет
- При count > 1 над предметом появляется спрайт-бейдж с числом

### Взятие (take)
- Raycaster пересекает меши предметов на полках (`allItemMeshes[]`)
- Активируется если инвентарь не полон (`!isFull()`) И прицел наведён на конкретный предмет
- Зелёная обводка (Game.Outline) только на предмете (не на стеллаже), подсказка "Взять {название} на ЛКМ"
- ЛКМ → забирает предметы bulk-ом: `addItemBulk(type, slot.count)` добавляет до maxStack за раз, `takeItemsFromShelf(slot, added)` уменьшает count слота на забранное количество; при count=0 удаляет меш
- Если инвентарь полон → уведомление "Инвентарь полон"

## Tool Panel Interaction
Настенная панель для инструментов. 3 типизированных крюка — каждый под конкретный инструмент.
- Панель стартует **пустой** — инструменты нужно покупать в магазине (первый каждого типа бесплатно)
- Два режима: **размещение** (place) и **взятие** (take)
- Блокируется аналогично стеллажам (isHoldingBox, hasBoxInteraction, etc.)

### Размещение (place)
- Raycaster пересекает все меши панели (`allPanelParts[]`)
- Активируется если в активном слоте инвентаря **инструмент** И его слот на панели свободен
- Зелёная обводка (Game.Outline) на всей панели, подсказка "Повесить на E"
- **E (KeyE)** → забирает из инвентаря, вешает на крюк (масштаб 2.5x, наклон rotation.z=0.15)

### Взятие (take)
- Raycaster пересекает меши инструментов на панели (`allPanelItemMeshes[]`)
- Зелёная обводка (Game.Outline) на инструменте, подсказка "Взять {название} на ЛКМ"
- ЛКМ → добавляет в инвентарь, убирает с крюка

## Furniture Interaction (Game.Furniture)

### Система перемещения мебели
- Вся мебель (кровати, стулья, стеллажи, касса) перемещается зажатием E
- Стеллажи регистрируются через `registerFixture()` с обратным вызовом `onMoved` для обновления слотов
- Raycaster из центра экрана (far=5), пересекает все группы мебели
- Приоритет определяется через `Game.Interaction` — ближайший объект побеждает
- При наведении: зелёная обводка (Game.Outline), подсказка "Зажми E — Переместить [название]"
- **Приоритет модулей**: когда модуль стеллажа активен (shelvesPlace/shelvesItems), но не имеет значимого взаимодействия (`hasInteraction()` = false), система переключается на мебельный модуль (fallback) — показывает зелёный outline и подсказку для перемещения
- **Условные raycaster-меши**: `shelvesPlace` возвращает меши только при наличии размещаемого расходника в руке. Это позволяет мебельному модулю обрабатывать перемещение в остальных случаях

### Подъём мебели (E)
- Если мебель занята (на ней лежит/сидит пациент) → уведомление "Нельзя переместить — предмет занят"
- При подъёме: collision box убирается из `collidables[]`, мебель следует за камерой (3 ед. вперёд, y=0.3)
- Мебель сохраняет исходный угол поворота (не вращается при переносе)

### Размещение мебели (E при переносе)
- Каждый кадр проверяется валидность размещения через AABB overlap с `collidables[]`
- **Зелёный outline** — можно поставить (Game.Outline, цвет 0x00ff44)
- **Красный outline** — нельзя поставить (Game.Outline, цвет 0xff2222)
- **Настенные предметы** (`wallMount: true`): привязываются к ближайшей стене, авто-поворот лицом в комнату, вращение колёсиком отключено; красный outline если стена не найдена
- При E:
  - Если размещение невалидно → уведомление "Нельзя разместить здесь"
  - Если валидно: мебель ставится на y=0, collision box возвращается в `collidables[]`
  - Если мебель вне здания → уведомление "Пока кровать/стул на улице — её нельзя использовать"
- Проверка "внутри здания": `x >= -7.8 && x <= 7.8 && z >= -11.8 && z <= -0.2`
- При размещении обновляется позиция слота пациента (`slot.pos = furniture.pos + slotOffset`)

### Слоты мебели
- Bed slotOffset: +1.0 по X (пациент стоит справа от кровати)
- Chair slotOffset: -1.0 по X (пациент стоит слева от стула)
- Слоты динамические — обновляются при каждом перемещении мебели

### Indoor/Outdoor статус
- Пациенты могут использовать только мебель внутри здания (`isIndoors === true`)
- Если есть мебель на улице, в попапе пациента показывается предупреждение: "Пока кровать/стул на улице — их нельзя использовать"
- Попап показывает только indoor кровати/стулья при подсчёте свободных мест

### Износ и ремонт кроватей (Game.Furniture + Game.Wrench)
- Каждая indoor-кровать имеет HP в диапазоне 0..10 (`BED_MAX_HP = 10`), стартовое значение 10
- После выписки или ухода пациента (HP=0) `Game.Furniture.decrementBedHp(slot)` уменьшает HP кровати на 1
- Визуальные состояния (меняется только каркас кровати, матрас/подушка не затрагиваются):
  - **HP 6-10** — норма, без изменений
  - **HP 1-5** — изношенная: 3 тёмные полосы-«трещины» (`0x222222`) лежат на раме, цвет каркаса слегка затемняется (×0.78)
  - **HP 0** — сломана: каркас ещё темнее (×0.55), боковая перекладина (rail) скрыта, к группе добавляются два тёмных «обломка» на полу рядом, кровать наклонена на ~3.4° по Z
- Сломанная (0 HP) кровать **не может** использоваться для новых пациентов (фильтруется по `isBedBroken(slot)`)
- В попапе пациента отображается предупреждение: "N кроватей сломано — почините ремонтным ключом" (одновременно с предупреждением об уличной мебели)

### Ремонтный ключ (Game.Wrench)
- Клавиша **R** переключает ключ (видимая first-person модель в правом-нижнем углу экрана крепится как ребёнок камеры, позиция ≈ (0.28, -0.22, -0.45))
- Модель: рукоять (CylinderGeometry 0.015×0.25) + тёмная накладка-грип + головка (BoxGeometry 0.08×0.03×0.05) с прорезью; материал `0x888899`
- Переключение блокируется: `!controls.isLocked`, открыт попап пациента/магазина/кассы, активна диагностика, перенос мебели
- **HP-бары над кроватями**: пока ключ экипирован И `controls.isLocked`, над каждой **indoor** кроватью появляется 3D-спрайт с сегментированным HP-баром (10 сегментов, Canvas 220×28, scale 1.1×0.14, y=1.4). Цвета: зелёный (HP 6-10), жёлтый (HP 4-5), красный (HP 1-3), тёмно-красный (HP 0). Заливка незанятых сегментов `rgba(255,255,255,0.10)`. При снятии ключа бары скрываются. У переносимой кровати бар тоже скрыт. Бары обновляются при `decrementBedHp` / `repairBed` / завершении hold-E ремонта и следуют за кроватью при перемещении
- Когда ключ экипирован и прицел наведён на кровать:
  - Подсказка "Кровать: N/10 HP — зажмите E для ремонта" (если HP > 0)
  - Подсказка "Кровать сломана (0/10) — зажмите E для ремонта" (если HP = 0)
- **Зажатие E** (3 секунды, общий hold-progress SVG-кольцом): по завершении вызывается `Game.Furniture.repairBed(slot)` — HP восстанавливается до 10, визуал возвращается к норме, уведомление "Кровать отремонтирована!" (зелёное)
- Пока ключ экипирован, обычный подъём мебели через hold-E на кроватях не выполняется (ветка ремонта имеет приоритет); стулья и другая мебель поднимаются как раньше

### Максимальная очередь
- Базовый максимум: 2 пациента
- Каждая купленная мебель (сверх начальных 5) увеличивает лимит на 1
- Формула: `maxQueue = min(2 + totalFurniture - 5, 10)`, минимум 2
- Начальное состояние: 2 кровати + 3 стула = 5 → maxQueue = 2


### Public API
- `setup(THREE, scene, camera, controls, collidables)` — инициализация
- `registerExisting(bedMeshes, chairMeshes)` — регистрация начальных кроватей/стульев из World
- `registerFixture(opts)` — регистрация доп. мебели (стеллажи, панель, касса и т.д.): `{type, group, collisionBox, onMoved, canPickUp, wallMount}`
- `updateCarried()` — обновление позиции переносимой мебели (каждый кадр, 60fps)
- `update(delta)` — обновление каждый 2-й кадр (interaction, hover, hold progress)
- `spawnFurniture(type)` — создание мебели в зоне доставки
- `getIndoorBeds()` / `getIndoorChairs()` — слоты для пациентов (только indoor)
- `getAllBeds()` / `getAllChairs()` — все слоты (для подсчёта очереди)
- `getOutdoorBedCount()` / `getOutdoorChairCount()` — количество outdoor мебели
- `isBedSlot(slot)` — проверка: слот принадлежит кровати
- `decrementBedHp(slot)` — уменьшить HP кровати на 1 (при выписке/уходе пациента)
- `repairBed(slot)` — восстановить HP кровати до 10 (hold-E с ремонтным ключом)
- `isBedBroken(slot)` → boolean — кровать со сломанным HP (<=0)
- `getBedHp(slot)` → number — текущее HP кровати (0..10)
- `getBrokenBedCount()` → number — количество сломанных indoor кроватей
- `BED_MAX_HP` → 10 — константа максимального HP
- `hasInteraction()` / `isCarrying()` — для click-блокировки между модулями

## Box Interaction

### На земле
- Raycaster из центра экрана (far=5) пересекает grounded + !pickedUp коробки (recursive)
- При наведении: зелёная обводка (Game.Outline), подсказка "ЛКМ — Взять: {название} (N шт.) | E — Поднять коробку"
- **ЛКМ** → взять препараты bulk-ом: `addItemBulk(type, remaining)` заполняет слот до maxStack, `remaining` уменьшается на взятое количество, метка обновляется (`updateBoxLabel()`), коробка остаётся на месте
- **E** → коробка берётся в руки (heldBox), следует за камерой
- Нельзя подобрать если уже держит коробку
- Если `remaining = 0` → коробка удаляется из сцены
- Если инвентарь полон → уведомление "Инвентарь полон"

### В руках (held box)
- **ЛКМ** → `Game.Inventory.addItemBulk(type, remaining)` — заполняет слот до maxStack, `remaining` уменьшается на взятое количество, метка обновляется (`updateBoxLabel()`)
- Если `remaining = 0` → коробка удаляется из сцены
- Если инвентарь полон → уведомление "Инвентарь полон"

### Throw Box (G при удержании коробки)
- Приоритет над дропом из инвентаря
- Коробка бросается вперёд от камеры (те же скорости: 4.0 forward, 2.0 up)
- Коробка сохраняет remaining и может быть подобрана снова

## Consumable Interaction

### Pickup
- Raycaster из центра экрана (far=5) пересекает grounded + !pickedUp предметы (recursive для Group)
- При наведении: зелёная обводка (Game.Outline), подсказка "Поднять на ЛКМ"
- ЛКМ → `Game.Inventory.addItem(type)`, удаление из сцены
- Если инвентарь полон → уведомление "Инвентарь полон"

### Drop (G / KeyG)
- Если держит коробку → бросает коробку (приоритет)
- Иначе: забирает предмет из активного слота инвентаря
- Создаёт mesh перед камерой (0.8 ед вперёд, -0.3 по Y)
- Бросает вперёд (скорость 4.0) и вверх (скорость 2.0)
- Предмет сталкивается со стенами, мебелью, полом через `collidables[]`
- Приземлившийся предмет можно снова подобрать
