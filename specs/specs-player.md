# EmergencyRND V2 — Игрок, управление, взаимодействие

## Player
- Стартовая позиция: `(0, 1.6, -10)` (за ресепшн-стойкой), лицом к улице (+Z)
- Камера: PerspectiveCamera, FOV 70, near 0.1, far 100
- `rotation.order = 'YXZ'` — совпадает с внутренним euler order PointerLockControls, предотвращает quaternion↔euler drift
- Начальный поворот: `camera.quaternion.setFromEuler(new Euler(0, PI, 0, 'YXZ'))` — лицом к входу/улице

## Controls
- **WASD** — движение (через `e.code`, работает на любой раскладке)
- **Мышь** — вращение камеры (Pointer Lock API, `pointerSpeed = 0.5`)
- **ЛКМ** — взаимодействие (пациент / лечение / подбор препарата / взять из коробки / взять со стеллажа / оплата на кассе)
- **E (KeyE)** — подобрать коробку в руки (не в инвентарь); поднять/поставить мебель (кровать/стул); положить на стеллаж
- **Q (KeyQ)** — открыть/закрыть магазин препаратов
- **G (KeyG)** — бросить коробку (если держит) или предмет из активного слота инвентаря (вперёд по камере с физикой)
- **1-6 (Digit1-Digit6)** — выбор слота инвентаря
- **Shift (ShiftLeft)** — ускорение (спринт): скорость `7.0` ед/сек вместо `4.0`
- **Space** — прыжок (начальная скорость `5.0`, гравитация `-12.0`); горизонтальное движение сохраняется в воздухе
- **ESC** — пауза (разблокировка курсора, показ overlay)
- Скорость движения: `4.0` ед/сек (обычная), `7.0` ед/сек (спринт)
- Высота камеры: `1.6` (базовая, меняется при прыжке)

## Mouse Delta Filter
- Capture-phase `mousemove` listener на `document`, срабатывает ДО PointerLockControls
- Блокирует события с `|movementX|` или `|movementY|` > 150 через `stopImmediatePropagation()`
- Предотвращает резкие прыжки/перевороты камеры от аномальных дельт браузера (баг Pointer Lock API)

## Controls Internal State
```js
_keys { forward, backward, left, right, sprint, jump }  // состояние клавиш
_raycaster                // для коллизий
_collisionOrigin          // Vector3, точка начала рейкаста (y=0.5, центр тела)
_savedQuat                // Quaternion|null, сохраняется при lock, восстанавливается в первом update
_forward, _right, _moveDir, _moveX, _moveZ  // pre-allocated Vector3, переиспользуются каждый кадр (без GC-давления)
_moveSpeed = 4.0
_sprintSpeed = 7.0
_collisionDistance = 0.4
_velocityY = 0            // вертикальная скорость (прыжок/падение)
_jumpSpeed = 5.0           // начальная скорость прыжка
_gravity = -12.0           // гравитация
_groundY = 1.6             // базовая высота камеры
_isGrounded = true         // на земле ли игрок
```

## Inventory System

### Data Model
```js
slots = [null, null, null, null, null, null]  // 6 слотов, значение = {type, count} или null
activeSlot = 0  // индекс 0-5
maxStack = 5    // макс. кол-во препаратов в одном слоте (базовое значение)
```

### Стакинг
- Препараты одного типа складываются в один слот (до `maxStack` штук)
- Инструменты НЕ стакаются — всегда занимают отдельный слот (count=1)
- `addItem(type)` для препаратов: сначала ищет существующий стак того же типа с `count < maxStack`, потом пустой слот
- `removeActive()` уменьшает count на 1; если count=0 — слот обнуляется
- На занятых слотах отображается бейдж `.inv-count` с количеством (если count > 1, только для препаратов)
- Базовый maxStack = 5, расширяется через апгрейд "Прокачка героя" в магазине

### UI (`#inventory-container` > `#inventory-active-name` + `#inventory-bar` + `#inventory-hints`)
- Контейнер `#inventory-container`: `fixed bottom: 16px`, по центру, z-index: 5, flex column
- `#inventory-active-name`: название активного предмета над баром (скрыто если слот пуст)
- `#inventory-bar`: 6 слотов (76x76px), rgba(0,0,0,0.5) фон, border
- Активный слот: золотая рамка (#ffcc00) + box-shadow
- Занятые слоты: canvas-иконки 128x128 (уникальные для каждого типа: блистер, флакон, коробка с крестом, фонендоскоп, молоток, риноскоп, чистое бельё, грязное бельё) — функция `generateIcon(type)` с кешированием
- Бейдж количества `.inv-count`: абсолютно позиционирован в правом верхнем углу слота, показывает count (только для препаратов с count > 1)
- Подпись 1-6 под каждым слотом
- Клавиши Digit1-Digit6 переключают activeSlot
- Поддерживает как препараты (`Game.Consumables.TYPES`), так и инструменты (`Game.Consumables.INSTRUMENT_TYPES`)
- `#inventory-hints`: строка подсказок под слотами
  - "Q — Магазин" — всегда видна
  - "G — Бросить" — видна только когда в активном слоте есть предмет
  - Стиль подсказок: крупный шрифт (0.9rem), яркий текст (85% opacity), жирный, тёмный фон

### Notification (`#notification`)
- z-index: 15, по центру
- Поддерживает цвет фона через параметр `color`
- Зелёный (`rgba(34, 139, 34, 0.85)`) для позитивных событий
- Красный (`rgba(200, 50, 50, 0.85)`) по умолчанию для негативных/нейтральных
- Появляется на 2 секунды, затем затухает (opacity transition)

### Inventory API
- `addItem(type)` → boolean (true если добавлено). Для препаратов: сначала стакает в существующий слот, потом в пустой
- `removeActive()` → type или null. Уменьшает count на 1, обнуляет слот при count=0
- `getActive()` → type активного слота или null
- `isFull()` → boolean
- `countType(type)` → number — суммарное количество предметов данного типа (сумма count по всем слотам)
- `setMaxStack(n)` → устанавливает максимальный размер стака
- `getMaxStack()` → текущий максимальный размер стака
- `showNotification(text, color)` → показать временное уведомление (color опционален, по умолчанию красный)

## Shop System

### Shop Popup (`#shop-popup`)
- z-index: 20, такой же стиль как `#patient-popup`
- Открывается на KeyQ, закрывается на KeyQ или кнопку ✕
- Блокируется если открыт попап пациента или кассы (`Game.Patients.isPopupOpen()`, `Game.Cashier.isPopupOpen()`)
- При открытии: `controls.unlock()`, обновляются счётчики предметов
- При закрытии: `controls.lock()` (кнопка закрытия — user gesture)
- Четыре таба: "Расходники", "Инструменты", "Мебель" и "Прокачка" (`#shop-tabs`)
- **Таб "Расходники"** (`#shop-tab-consumables`):
  - 4 товара: Стрепсилс, Обезболивающее, Антигистаминное, Постельное бельё
  - Цветная иконка-кружок + кнопка "Купить — $10"
  - Коробка (10 штук) стоит $10, списывается через `Game.Cashier.spend(10)`
  - При покупке спавнится коробка в зоне доставки (`Game.Consumables.spawnBoxInDeliveryZone(type)`)
- **Таб "Инструменты"** (`#shop-tab-instruments`):
  - 3 инструмента: Фонендоскоп, Рефлекс-молоток, Риноскоп
  - Цена $1, покупка → одиночный предмет в зоне доставки (`Game.Consumables.spawnInstrumentInDeliveryZone(type)`)
  - Инструменты многоразовые (не расходуются при использовании)
- **Таб "Мебель"** (`#shop-tab-furniture`):
  - 2 товара: Кровать ($25), Стул ($15)
  - При покупке мебель появляется в зоне доставки (`Game.Furniture.spawnFurniture(type)`)
  - Каждая покупка увеличивает максимальную очередь на 1 (до максимума 10)
  - Мебель на улице недоступна для пациентов — нужно перенести внутрь здания
- **Таб "Прокачка"** (`#shop-tab-upgrades`):
  - Апгрейд "Больше препаратов" — увеличивает maxStack инвентаря
  - 3 уровня, покупаются последовательно (нельзя купить ур. 2 без ур. 1):
    - Уровень 1: 10 шт. в слоте — $50
    - Уровень 2: 15 шт. в слоте — $100
    - Уровень 3: 20 шт. в слоте — $200
  - Кнопка доступного уровня: "Купить — $N"
  - Кнопка купленного уровня: "Куплено ✓" (disabled, серая)
  - Кнопка заблокированного уровня: "Сначала купите ур. N" (disabled, серая)
  - При покупке: `Game.Cashier.spend(price)`, `Game.Inventory.setMaxStack(stack)`, зелёное уведомление
  - Кнопки обновляются при открытии магазина и после каждой покупки (`refreshUpgradeButtons()`)
  - Иконка: оранжевый кружок (#e8a030)
  - Описание уровня: мелкий текст `.shop-upgrade-desc` под названием
- Если баланс недостаточен → уведомление "Недостаточно средств!"
- Блокируется также при `Game.Diagnostics.isActive()`
- У каждого товара отображается счётчик "(есть: N)". Обновляется при открытии и после каждой покупки

## Shelf Interaction
- Два режима: **размещение** (place) и **взятие** (take)
- Блокируется если игрок держит коробку (`isHoldingBox()`) или наведён на коробку (`hasBoxInteraction()`)

### Размещение (place)
- Raycaster пересекает все меши стеллажей (`allShelfParts[]`)
- Активируется если в активном слоте инвентаря есть предмет И на стеллаже есть доступный слот
- Стекирование: до 10 предметов одного типа на один слот (`MAX_SHELF_STACK = 10`)
- При размещении: сначала ищется слот с тем же типом (для стека), затем пустой слот
- Зелёный emissive на всём стеллаже, подсказка "Положить на E"
- **E (KeyE)** → забирает из инвентаря, создаёт 3D модель на полке (или увеличивает count при стекировании)
- Инструменты масштабируются на полке (`INSTRUMENT_SHELF_SCALE = 2.8` от базового размера)
- При count > 1 над предметом появляется спрайт-бейдж с числом

### Взятие (take)
- Raycaster пересекает меши предметов на полках (`allItemMeshes[]`)
- Активируется если инвентарь не полон (`!isFull()`) И прицел наведён на конкретный предмет
- Зелёный emissive только на предмете (не на стеллаже), подсказка "Взять {название} на ЛКМ"
- ЛКМ → добавляет предмет в инвентарь, уменьшает count слота; при count=0 удаляет меш
- Если инвентарь полон → уведомление "Инвентарь полон"

## Furniture Interaction (Game.Furniture)

### Система перемещения мебели
- Мебель (кровати и стулья) можно перемещать с помощью клавиши E
- Raycaster из центра экрана (far=5), пересекает все группы мебели
- Приоритет: выше коробок, ниже пациентов
- При наведении: зелёный emissive (0x00ff44, intensity 0.35), подсказка "Нажми E чтобы переместить"

### Подъём мебели (E)
- Если мебель занята (на ней лежит/сидит пациент) → уведомление "Нельзя переместить — предмет занят"
- При подъёме: collision box убирается из `collidables[]`, мебель следует за камерой (3 ед. вперёд, y=0.3)
- Мебель сохраняет исходный угол поворота (не вращается при переносе)

### Размещение мебели (E при переносе)
- Каждый кадр проверяется валидность размещения через AABB overlap с `collidables[]`
- **Зелёный outline** — можно поставить (emissive 0x00ff44, intensity 0.45)
- **Красный outline** — нельзя поставить (emissive 0xff2222, intensity 0.45)
- При E:
  - Если размещение невалидно → уведомление "Нельзя разместить здесь"
  - Если валидно: мебель ставится на y=0, collision box возвращается в `collidables[]`
  - Если мебель вне здания → уведомление "Пока кровать/стул на улице — её нельзя использовать"
- Проверка "внутри здания": `x >= -7.8 && x <= 7.8 && z >= -11.8 && z <= -0.2`
- При размещении обновляется позиция слота пациента (`slot.pos = furniture.pos + slotOffset`)

### Слоты мебели
- Bed slotOffset: +1.0 по X (пациент стоит справа от кровати)
- Chair slotOffset: -1.0 по X (пациент стоит слева от стула)
- Слоты динамические — обновляются при каждом перемещении мебели

### Indoor/Outdoor статус
- Пациенты могут использовать только мебель внутри здания (`isIndoors === true`)
- Если есть мебель на улице, в попапе пациента показывается предупреждение: "Пока кровать/стул на улице — их нельзя использовать"
- Попап показывает только indoor кровати/стулья при подсчёте свободных мест

### Грязное бельё на кроватях
- После выписки или ухода пациента (HP=0) кровать становится грязной (`isDirty = true`)
- Грязная кровать отмечается визуальным overlay (полупрозрачный коричнево-жёлтый слой на матрасе)
- Грязная кровать **не может** использоваться для новых пациентов (фильтруется в `openPopup()` и при назначении)
- В попапе пациента отображается предупреждение: "Грязное бельё на N кроватях — замените бельё" (одновременно с предупреждением об уличной мебели)

### Замена белья (ЛКМ на грязную кровать)
- Если игрок наводит прицел на грязную кровать с чистым бельём (`linen_clean`) в активном слоте:
  - Подсказка: "ЛКМ — Заменить бельё"
  - ЛКМ: чистое бельё расходуется, кровать становится чистой, грязное бельё (`linen_dirty`) попадает в инвентарь
  - Если инвентарь полон — грязное бельё падает на пол через `Game.Consumables.dropFromPlayer('linen_dirty')`
  - Уведомление: "Бельё заменено!" (зелёное)
- Если без белья: подсказка "Нужно чистое бельё для замены"

### Стиральная машина (Game.WashingMachine)
- Позиция: (5.5, 0, -10.5), северо-восточный угол здания
- При наведении: зелёный emissive highlight (как у мебели)
- **ЛКМ**: загрузить грязное бельё из активного слота (`linen_dirty`), максимум 5 шт. за стирку
- **E**: запустить стирку (если загружено > 0 и не стирается)
- Стирка длится 20 секунд, барабан вращается, прогресс-бар заполняется
- По завершении: чистое бельё (`linen_clean`) спавнится на крышке машины (через `Game.Consumables.spawnAtPosition`)
- Динамические подсказки: меняются в зависимости от состояния (пусто/загружено/стирка/есть dirty в руках)
- Индикатор состояния: зелёный (свободна), оранжевый (загружено), жёлтый (стирка)
- Блокируется при: открыт попап, магазин, касса, диагностика, перенос мебели

### Максимальная очередь
- Базовый максимум: 2 пациента
- Каждая купленная мебель (сверх начальных 5) увеличивает лимит на 1
- Формула: `maxQueue = min(2 + totalFurniture - 5, 10)`, минимум 2
- Начальное состояние: 2 кровати + 3 стула = 5 → maxQueue = 2

### Internal State
```js
furnitureItems[]       // все предметы мебели [{type, group, collisionBox, slot, isIndoors, isDirty, dirtyOverlay}]
carriedFurniture       // переносимый предмет (или null)
hoveredFurniture       // предмет под прицелом (или null)
canPlaceCurrent        // bool — можно ли поставить в текущей позиции
lastCarryColor         // 'green'|'red'|null — текущий цвет outline переносимой мебели
```

### Public API
- `setup(THREE, scene, camera, controls, collidables)` — инициализация
- `registerExisting(bedMeshes, chairMeshes)` — регистрация начальных кроватей/стульев из World
- `update(delta)` — обновление каждый кадр
- `spawnFurniture(type)` — создание мебели в зоне доставки
- `getIndoorBeds()` / `getIndoorChairs()` — слоты для пациентов (только indoor)
- `getAllBeds()` / `getAllChairs()` — все слоты (для подсчёта очереди)
- `getOutdoorBedCount()` / `getOutdoorChairCount()` — количество outdoor мебели
- `isBedSlot(slot)` — проверка: слот принадлежит кровати
- `markBedDirty(slot)` / `markBedClean(slot)` — управление грязным бельём
- `isBedDirty(slot)` → boolean — грязная ли кровать
- `getDirtyBedCount()` → number — кол-во грязных indoor кроватей
- `tryLinenReplace()` → boolean — попытка замены белья по ЛКМ на грязную кровать
- `hasInteraction()` / `isCarrying()` — для приоритета взаимодействий

## Box Interaction

### На земле
- Raycaster из центра экрана (far=5) пересекает grounded + !pickedUp коробки (recursive)
- При наведении: зелёный emissive (надписи пропускаются — `userData.isLabel`), подсказка "ЛКМ — Взять препарат (N шт.) | E — Поднять коробку"
- **ЛКМ** → взять 1 препарат из коробки в инвентарь (`remaining--`), коробка остаётся на месте
- **E** → коробка берётся в руки (heldBox), следует за камерой
- Нельзя подобрать если уже держит коробку
- Если `remaining = 0` → коробка удаляется из сцены
- Если инвентарь полон → уведомление "Инвентарь полон"

### В руках (held box)
- **ЛКМ** → `Game.Inventory.addItem(type)`, `remaining--`
- Если `remaining = 0` → коробка удаляется из сцены
- Если инвентарь полон → уведомление "Инвентарь полон"

### Throw Box (G при удержании коробки)
- Приоритет над дропом из инвентаря
- Коробка бросается вперёд от камеры (те же скорости: 4.0 forward, 2.0 up)
- Коробка сохраняет remaining и может быть подобрана снова

## Consumable Interaction

### Pickup
- Raycaster из центра экрана (far=5) пересекает grounded + !pickedUp предметы (recursive для Group)
- При наведении: зелёный emissive (0x00ff44, intensity 0.35), подсказка "Поднять на ЛКМ"
- ЛКМ → `Game.Inventory.addItem(type)`, удаление из сцены
- Если инвентарь полон → уведомление "Инвентарь полон"

### Drop (G / KeyG)
- Если держит коробку → бросает коробку (приоритет)
- Иначе: забирает предмет из активного слота инвентаря
- Создаёт mesh перед камерой (0.8 ед вперёд, -0.3 по Y)
- Бросает вперёд (скорость 4.0) и вверх (скорость 2.0)
- Предмет сталкивается со стенами, мебелью, полом через `collidables[]`
- Приземлившийся предмет можно снова подобрать
